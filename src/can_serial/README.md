# can_serial

can通信实现  
波特率1000000  
字节存储使用小端模式

## include
包含can和ros2头文件声明和can定义  
`CanSerialCore.hpp` can头文件声明  
`CanSerialCore.cpp` can函数定义  
`CanSerialNode.hpp` ros2 node头文件声明

## src
ros2 node代码放在此目录下的 `CanSerialNode.cpp` 文件中  

can通信的接收处理函数绑定必须在ros2创建完发布者之后，不然会出现 Segmentation Fault

## CAN 硬件状态码

`/can_hardware_state` 话题发布 `std_msgs::msg::UInt8`，状态码含义如下：

| 状态码 | 状态名称 | 含义 | 常见原因 |
|--------|----------|------|----------|
| 0 | ACTIVE | 正常通信 | CAN 总线正常工作，控制器处于主动错误态 |
| 1 | WARNING | 控制器警告 | 总线上有间歇性噪声/干扰，错误计数超过 96，仍可正常收发 |
| 2 | PASSIVE | 被动错误 | 持续通信故障，错误计数超过 127，只能被动接收，发送需等待 |
| 3 | BUS_OFF | 总线关闭 | 控制器因错误过多被踢出总线，**完全无法通信**，需重新初始化 |
| 4 | IF_DOWN | 网卡已 down | CAN 网卡被关闭（`ip link set can0 down`）或未配置 |
| 5 | NO_SLAVE | 下位机未连接 | CAN 控制器正常，但总线上无设备响应（ACK 错误）或从未收到下位机数据 |

### 状态转换路径

```
ACTIVE(0) ──错误累积──▶ WARNING(1) ──错误累积──▶ PASSIVE(2) ──错误累积──▶ BUS_OFF(3)
    ▲                      │                      │                      │
    └──────────────────────┴──────────────────────┴─── 恢复正常需重新初始化 ──┘

    │
    ├── 探测帧无 ACK / 从未收到数据 ──▶ NO_SLAVE(5)
    │                                       │
    │                          收到下位机有效数据帧
    │                                       │
    │◀──────────────────────────────────────┘

正常                        │  IF_DOWN(4)  ←── ip link set can0 down / 网卡异常
                            │      ▲
                            └──────┘  可互相切换
```

### 常见故障场景

#### 1. 下位机未连接 / 未上电
- **现象：** 节点启动后显示 NO_SLAVE(5)，探测帧无 ACK 响应
- **检测方式：** 每 500ms 发送探测帧（不受 `calibrated` 限制），监听 `CAN_ERR_ACK` 错误帧
- **状态变化：** 直接为 NO_SLAVE(5)
- **恢复：** 连接下位机并上电，收到有效数据帧后自动恢复为 ACTIVE(0)

#### 2. 下位机断电 / CAN 线缆断开（运行中）
- **现象：** 上位机持续发送 CAN 帧，但无节点应答（ACK 错误）
- **状态变化：** ACTIVE(0) → NO_SLAVE(5)；如果持续发送可能进一步进入 WARNING(1) → PASSIVE(2) → BUS_OFF(3)
- **恢复：** 重新给下位机上电

#### 3. CAN 总线终端电阻缺失或接线不良
- **现象：** 通信时断时续，状态在 ACTIVE 和 WARNING 之间反复
- **状态变化：** ACTIVE(0) ⇄ WARNING(1)
- **恢复：** 检查总线两端是否正确接入 120Ω 终端电阻

#### 4. CAN 网卡未配置或已关闭
- **现象：** 节点启动后立即发布 IF_DOWN(4)
- **状态变化：** 直接为 IF_DOWN(4)
- **恢复：** `sudo ip link set can0 up type can bitrate 1000000`

#### 5. 总线波特率不匹配
- **现象：** 上位机和下位机波特率不一致，所有帧 CRC 错误
- **状态变化：** 快速进入 BUS_OFF(3)
- **恢复：** 确认双方波特率一致（当前为 1000000），重新配置 CAN 网卡
